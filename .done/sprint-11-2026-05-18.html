Destaque · Feature 01

Memória cross-channel ligou
site e WhatsApp sob o
mesmo cliente.

Usuário que conversou no site há 5 dias e volta hoje pelo WhatsApp é reconhecido automaticamente. Agente retoma de onde parou em vez de refazer a jornada inteira — diferencial inexistente nas administradoras tradicionais que ainda operam PDF + corretor + e-mail.

  • Reconhecimento por telefone (WhatsApp) ou cookie (site anônimo)
  • Memória de simulações, objeções, faixa de crédito e prazo discutidos
  • Detector de retorno após N dias com adaptação automática de tom
  • Fallback silencioso se a camada cair — chat nunca trava

Destaque · Feature 02

Framework de regressão em 3 camadas blinda
o agente contra reincidência.

Cada bug agentic reportado vira cassette determinístico. Mexer no prompt, na tool ou no seed do agente quebra teste antes do push — não em produção pelo cliente.

Camada 1 — Estrutural

Lê source do prompt e builder. Pega tool faltando, regra ausente, seed quebrado. ~3,5s em todo PR + pre-commit.

64 arquivos test.ts <1s/arquivo

Camada 2 — Cassettes

MockLanguageModel reproduz comportamento exato observado em bug real. Pega meta-narrativa, tools que prometem e não disparam.

23 cassettes <500ms suite

Camada 3 — LLM Judge

user-bot (Haiku) ↔ agent real (Sonnet) conversam de verdade. Pega drift de modelo. Roda nightly, não bloqueia merge.

Cron noturno Haiku + Sonnet
