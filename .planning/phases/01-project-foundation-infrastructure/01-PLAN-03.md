### Task 2: Create multi-stage Dockerfile

<read_first>
- package.json
- next.config.ts (verify standalone output is set)
- .planning/phases/01-project-foundation-infrastructure/01-RESEARCH.md (Docker section)
</read_first>

<action>
Create `Dockerfile` at project root:

```dockerfile
FROM node:22-alpine AS base

# Install dependencies
FROM base AS deps
WORKDIR /app
COPY package.json package-lock.json ./
RUN npm ci

# Build the application
FROM base AS builder
WORKDIR /app
COPY --from=deps /app/node_modules ./node_modules
COPY . .
RUN npm run build

# Production image
FROM base AS runner
WORKDIR /app
ENV NODE_ENV=production

RUN addgroup --system --gid 1001 nodejs
RUN adduser --system --uid 1001 nextjs

COPY --from=builder /app/public ./public
COPY --from=builder --chown=nextjs:nodejs /app/.next/standalone ./
COPY --from=builder --chown=nextjs:nodejs /app/.next/static ./.next/static

USER nextjs
EXPOSE 3000
ENV PORT=3000
ENV HOSTNAME="0.0.0.0"

CMD ["node", "server.js"]
```

Key design decisions:
- `node:22-alpine` for minimal image size (Node 22 is LTS)
- 3-stage build: deps -> builder -> runner
- `npm ci` for deterministic dependency installation
- Non-root user (`nextjs:nodejs`) for security
- Only standalone output + public + static copied to final image
</action>

<acceptance_criteria>

- `Dockerfile` contains `FROM node:22-alpine AS base`
- `Dockerfile` contains `npm ci`
- `Dockerfile` contains `npm run build`
- `Dockerfile` contains `USER nextjs`
- `Dockerfile` contains `CMD ["node", "server.js"]`
- `Dockerfile` contains 3 stages: `deps`, `builder`, `runner`

### Task 3: Update docker-compose.yml with app service

<read_first>
- docker-compose.yml
</read_first>

<action>
Update the existing `docker-compose.yml` (created in Plan 02 Task 1) to add the app service while keeping the existing db service:

```yaml
services:
  app:
    build: .
    ports:
      - "3000:3000"
    environment:
      - DATABASE_URL=postgresql://postgres:postgres@db:5432/aja_agora
    depends_on:
      db:
        condition: service_healthy

  db:
    image: postgres:16-alpine
    ports:
      - "5432:5432"
    environment:
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: postgres
      POSTGRES_DB: aja_agora
    volumes:
      - pgdata:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U postgres"]
      interval: 5s
      timeout: 5s
      retries: 5

volumes:
  pgdata:
```

Note: The app service uses `db` as the hostname (Docker Compose networking), not `localhost`.
</action>

<acceptance_criteria>
- `docker-compose.yml` contains `app:` service
- `docker-compose.yml` contains `build: .`
- `docker-compose.yml` contains `DATABASE_URL=postgresql://postgres:postgres@db:5432/aja_agora`
- `docker-compose.yml` contains `depends_on` with `condition: service_healthy`
- `docker-compose.yml` still contains `db:` service with `postgres:16-alpine`
</acceptance_criteria>

### Task 4: Build Docker image and verify size

<read_first>
- Dockerfile
- .dockerignore
</read_first>

<action>
Build the Docker image:

```bash
docker build -t aja-agora .
```

Check image size using a parseable format:

```bash
docker images aja-agora --format "{{.Size}}"
```

The image MUST be under 500MB. Expected size is 150-250MB with Alpine + standalone output.

If the image exceeds 500MB:
1. Check `.dockerignore` is being applied (rebuild with `--no-cache`)
2. Verify `output: "standalone"` is in `next.config.ts`
3. Check for unexpected large dependencies in `package.json`
</action>

<acceptance_criteria>
- `docker build -t aja-agora .` completes successfully
- Image size is under 500MB
</acceptance_criteria>

### Task 5: Verify full-stack docker compose up

<read_first>
- docker-compose.yml
</read_first>

<action>
Start the full stack:

```bash
docker compose up -d
```

Wait for services to be healthy, then verify:

```bash
docker compose ps
curl http://localhost:3000
```

Both `app` and `db` services should be running. The curl command should return HTML from the Next.js app.

Clean up:

```bash
docker compose down
```
</action>

<acceptance_criteria>
- `docker compose up -d` starts both services without errors
- `docker compose ps` shows both `app` and `db` as running
- `curl http://localhost:3000` returns HTML content
- `docker compose down` stops cleanly
</acceptance_criteria>

<threat_model>
## Threat Model (ASVS L1)

| Threat | Severity | Mitigation |
|--------|----------|------------|
| Container running as root | High | Dockerfile creates non-root user `nextjs` with UID 1001; `USER nextjs` directive before CMD |
| Secrets baked into Docker image | High | `.dockerignore` excludes `.env*.local`; DATABASE_URL passed via environment variable in docker-compose.yml |
| Unpatched base image vulnerabilities | Med | Using `node:22-alpine` (minimal attack surface); rebuild periodically to pick up Alpine security patches |
| Source code exposed in image | Low | Multi-stage build: only `standalone` output, `public/`, and `.next/static/` are in the final image; no source code |
| PostgreSQL exposed on host port 5432 | Low | Dev-only configuration; production will use internal Docker networking without port exposure |
</threat_model>
