<action>
Create `src/components/chat/artifact-renderer.tsx`:

```typescript
// src/components/chat/artifact-renderer.tsx
import type { Artifact } from "@/lib/chat/types";
import { GroupCard } from "./artifacts/group-card";
import { ComparisonTable } from "./artifacts/comparison-table";
import { SimulationResult } from "./artifacts/simulation-result";
import type { ComponentType } from "react";

const ARTIFACT_COMPONENTS: Record<string, ComponentType<{ payload: any }>> = {
  group_card: GroupCard,
  comparison_table: ComparisonTable,
  simulation_result: SimulationResult,
};

export function ArtifactRenderer({ artifact }: { artifact: Artifact }) {
  const Component = ARTIFACT_COMPONENTS[artifact.type];
  if (!Component) {
    console.warn(`Unknown artifact type: ${artifact.type}`);
    return null;
  }
  return <Component payload={artifact.payload} />;
}
```

This is a simple type-dispatch — extensible for future artifact types (recommendation_card, lead_form) by adding entries to the map.
</action>

<threat_model>
## Threat Model (ASVS L1)
| Threat | Severity | Mitigation |
|--------|----------|------------|
| XSS via artifact payload fields (administradora name, etc.) | High | All values rendered as React text nodes (auto-escaped). No dangerouslySetInnerHTML. Payload is typed — no arbitrary HTML fields. |
| Misleading financial data injection | Medium | Presentation tools use Zod-validated inputs. Values come from adapter (mock or real API), not from user input. Agent can't fabricate values outside tool results. |
| Missing/undefined payload fields causing crash | Medium | Components should handle missing fields gracefully — show "—" or fallback. TypeScript enforces shape at compile time. |
| Clickjacking on GroupCard | Low | Cards are within the app's own page. No external iframes. CSP headers (future Phase) will prevent embedding. |
| Large comparison table (100+ rows) | Low | Agent practically limited by maxTurns=5 and context size. Comparison tables will have 2-5 rows. No pagination needed for MVP. |
</threat_model>
