// Wrap the message bubble:
<motion.div
  initial={{ opacity: 0, y: 12 }}
  animate={{ opacity: 1, y: 0 }}

Key decisions:

```typescript
// If not new, skip initial animation
const animationProps = isNew
  ? { initial: { opacity: 0, y: 12 }, animate: { opacity: 1, y: 0 } }
  : { initial: false, animate: { opacity: 1, y: 0 } };
```

CRITICAL: Import from `motion/react`, NOT `framer-motion`.

Edit the artifact rendering section in `src/components/chat/chat-message.tsx` (where artifacts are mapped) to wrap each artifact in a `motion.div` with `AnimatePresence` and staggered entry:

```typescript
import { motion, AnimatePresence } from "motion/react";

// In the artifacts rendering section:
<AnimatePresence mode="popLayout">
  {message.artifacts.map((artifact, i) => (
    <motion.div
      key={artifact.id}
      layout
      initial={{ opacity: 0, scale: 0.96, y: 8 }}
      animate={{ opacity: 1, scale: 1, y: 0 }}
      exit={{ opacity: 0, scale: 0.96 }}
      transition={{
        type: "spring",
        stiffness: 400,
        damping: 25,
        delay: i * 0.06,
      }}
    >
      <ArtifactRenderer artifact={artifact} />
    </motion.div>
  ))}
</AnimatePresence>
```

Key decisions:
- `mode="popLayout"` — elements pop out of layout flow during exit (prevents layout jumps)
- `layout` prop on each artifact — smooth repositioning if artifacts are added/removed
- `delay: i * 0.06` — 60ms stagger between consecutive artifacts (subtle cascade)
- `scale: 0.96` — very subtle scale, not dramatic
- Exit animation only matters if artifacts can be removed (edge case for future, but harmless to include)
</action>

### Task 4: Add prefers-reduced-motion support

<read_first>
- src/components/chat/chat-message.tsx (message animations)
- src/components/chat/artifacts/group-card.tsx (card animations)
- src/app/globals.css (global styles)
</read_first>

<action>
Add a CSS media query and a React hook to respect `prefers-reduced-motion`:

1. **Global CSS** — add to `src/app/globals.css`:
   ```css
   @media (prefers-reduced-motion: reduce) {
     *, *::before, *::after {
       animation-duration: 0.01ms !important;
       animation-iteration-count: 1 !important;
       transition-duration: 0.01ms !important;
     }
   }
   ```

2. **React hook** — create `src/lib/hooks/use-reduced-motion.ts`:
   ```typescript
   "use client";

   import { useEffect, useState } from "react";

   export function useReducedMotion(): boolean {
     const [prefersReduced, setPrefersReduced] = useState(false);

     useEffect(() => {
       const mql = window.matchMedia("(prefers-reduced-motion: reduce)");
       setPrefersReduced(mql.matches);

       const handler = (event: MediaQueryListEvent) => {
         setPrefersReduced(event.matches);
       };

       mql.addEventListener("change", handler);
       return () => mql.removeEventListener("change", handler);
     }, []);

     return prefersReduced;
   }
   ```

3. **Use in message animations** — conditionally skip animations when reduced motion is preferred. In components that use `motion.div`, conditionally set `initial={false}` and skip spring transitions when `prefersReduced` is true.

   Motion v12 partially respects the CSS media query automatically for CSS-based animations, but the JS-driven spring animations need explicit handling.
</action>

<threat_model>
## Threat Model (ASVS L1)
| Threat | Severity | Mitigation |
|--------|----------|------------|
| Animation jank on low-end devices | Medium | Only animate opacity and transform (GPU-composited). High stiffness = short animation duration. prefers-reduced-motion disables animations entirely. |
| Error message reveals internal server details | Medium | Error messages are user-facing Portuguese strings. Raw server errors are logged to console, not shown to user. |
| Retry button abuse (rapid clicking) | Low | isStreaming guard prevents concurrent retries. Backend rate limiting provides server-side protection. |
| CSS overflow-x-hidden hiding important content | Low | Only applied at root level to prevent horizontal scroll. Content uses responsive widths and horizontal scroll within containers (ComparisonTable). |
| visualViewport API not supported on older browsers | Low | Check `window.visualViewport` existence before attaching listeners. Graceful degradation — old browsers just don't get keyboard-aware scrolling. |
</threat_model>
