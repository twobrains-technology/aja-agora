## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding |
| sm | 8px | Compact element spacing, gap between streaming dots |
| md | 16px | Default element spacing, message bubble padding, card internal padding |
| lg | 24px | Gap between messages, card section padding |

| 2xl | 48px | Header height on mobile |

Exceptions:
- Touch targets minimum 44px (Apple HIG) — applies to send button, GroupCard tap area, any interactive element
- Chat input fixed container uses `env(safe-area-inset-bottom)` additive padding on notch devices

## Typography

| Role | Size | Weight | Line Height | Font | Usage |
|------|------|--------|-------------|------|-------|
| Body | 16px | 400 (regular) | 1.5 | Geist Sans | Message text, descriptions, general content |
| Label | 14px | 500 (medium) | 1.4 | Geist Sans | Category badges, metadata labels, timestamps, table headers |
| Heading | 20px | 600 (semibold) | 1.2 | Geist Sans | Card titles, section headers within artifacts |
| Financial | 24px | 700 (bold) | 1.2 | Geist Mono | Primary financial values — monthly parcela, credit amount, total cost |

**Rationale:** 4 sizes only. Geist Mono for financial numbers creates visual distinction and communicates precision. 16px body is the minimum readable size on mobile at arm's length.

**Restrictions:**
- No font size below 14px anywhere in the interface
- Financial values (R$ amounts, percentages, month counts) always use `font-mono` class

## Color

The current `globals.css` uses the default neutral (gray) shadcn theme. Phase 3 introduces brand color to `--primary` and `--accent` to create a trustworthy-fintech-but-not-boring identity for "Aja Agora."

### Brand Color: Teal/Emerald

Teal communicates trust, stability, and modernity — used by fintechs like Nubank's competitor Creditas, and banking apps globally. It avoids the coldness of pure blue and the boredom of corporate navy.

| Role | Token | Value (Light) | Value (Dark) | Usage |
|------|-------|---------------|--------------|-------|
| Dominant (60%) | `--background` | `oklch(0.985 0 0)` (warm white) | `oklch(0.145 0.005 260)` (near-black with hint of blue) | Page background, chat area |
| Secondary (30%) | `--card` | `oklch(1 0 0)` (white) | `oklch(0.205 0.005 260)` | Message bubbles (assistant), artifact cards, header |
| User bubble | `--primary` | `oklch(0.45 0.16 168)` (deep teal) | `oklch(0.55 0.16 168)` (lighter teal) | User message bubbles, send button filled, primary CTA |
| User bubble text | `--primary-foreground` | `oklch(0.985 0 0)` (white) | `oklch(0.985 0 0)` (white) | Text on user bubbles and primary buttons |
| Accent (10%) | `--accent` | `oklch(0.75 0.12 168)` (light teal) | `oklch(0.35 0.12 168)` | Highlight on best-option in ComparisonTable, streaming dots, hover states |
| Destructive | `--destructive` | `oklch(0.577 0.245 27.325)` (red) | `oklch(0.704 0.191 22.216)` | Error messages only |
| Muted | `--muted` | `oklch(0.97 0 0)` | `oklch(0.269 0 0)` | Assistant bubble background, disabled states |
| Border | `--border` | `oklch(0.922 0 0)` | `oklch(1 0 0 / 10%)` | Card borders, separator lines |

### Accent Reserved For (explicit list)

- Send button (filled primary)
- User message bubble background
- "Best option" highlight row/border in ComparisonTable
- Streaming dots animation color
- Interactive card hover border
- GroupCard "Simular" action text

**Accent is NOT used for:** general text, borders, backgrounds, non-interactive elements.

### Assistant vs User Bubble Colors

| Element | Light Mode | Dark Mode |
|---------|-----------|-----------|
| User bubble BG | `--primary` (deep teal) | `--primary` (lighter teal) |
| User bubble text | `--primary-foreground` (white) | `--primary-foreground` (white) |
| Assistant bubble BG | `--muted` (gray-50) | `--muted` (gray-800) |
| Assistant bubble text | `--foreground` (near-black) | `--foreground` (near-white) |

## Interaction Patterns

### Chat Input

| Behavior | Specification |
|----------|---------------|
| Input type | Auto-resize `<textarea>` with 1 row default, max 5 rows (120px) |
| Submit | Enter key (desktop), Send button (mobile + desktop). Shift+Enter for newline |
| Disabled state | During streaming — input grayed out, send button shows streaming dots |
| Placeholder | "Diga o que voce quer realizar..." |
| Focus | Auto-focus on page load. Re-focus after message sent |

### Message List

| Behavior | Specification |
|----------|---------------|
| Scroll | Auto-scroll to bottom on new messages when user is at bottom |
| Manual scroll | If user scrolls up, stop auto-scroll. Show "novas mensagens" pill to jump back |
| Streaming | Always auto-scroll during active streaming (user hasn't intentionally scrolled up) |
| Bottom sentinel | IntersectionObserver on invisible div at list bottom, threshold 0.5 |

### GroupCard Interaction

| Behavior | Specification |
|----------|---------------|
| Tap/Click | Triggers `get_group_details` — sends user message like "Me conta mais sobre esse grupo" with groupId context |
| Hover (desktop) | `scale: 1.02` with spring physics, border transitions to accent |
| Press (mobile) | `scale: 0.98` with spring physics for press feedback |
| Content | Category badge (top-left), credit value (heading, mono font), monthly parcela (financial, mono font), admin rate + term (label text) |

### ComparisonTable Interaction

| Behavior | Specification |
|----------|---------------|
| Scroll | Horizontal scroll on mobile with sticky first column (group identifier) |
| Best option | Row highlighted with `--accent` left border (4px) and subtle background tint |
| Columns | Group name, Credit (R$), Parcela (R$/mes), Taxa admin (%), Prazo (meses) |
| Tap row | Same behavior as GroupCard tap — request details |

### SimulationResult Interaction

| Behavior | Specification |
|----------|---------------|
| Layout | Monthly parcela in large financial typography at top. Below: 2-column grid with rate, reserve fund, insurance, total cost, term |
| No direct interaction | Display-only artifact. User can ask follow-up questions naturally in chat |

### Streaming Indicator

| Behavior | Specification |
|----------|---------------|
| Visual | 3 circles (8px diameter), `--muted-foreground` color, pulsing opacity 0.3 to 1.0 |
| Timing | 1.2s loop, 0.2s stagger between dots |
| Position | Inside an assistant bubble, left-aligned, appears while `isStreaming === true` and no text has arrived yet |
| Transition | Fade out when first text-delta arrives, message text fades in |

## Animation Contract (Motion v12)

All animations use `motion/react` imports. Never `framer-motion`.

| Element | Animation | Spring Config | Notes |
|---------|-----------|---------------|-------|
| New message (enter) | `opacity: 0→1, y: 20→0` | `stiffness: 300, damping: 30` | Only animate latest message, not all |
| Artifact card (enter) | `opacity: 0→1, scale: 0.95→1, y: 10→0` | `stiffness: 400, damping: 25` | Stagger: `delay: i * 0.05` for multiple cards |
| Artifact card (exit) | `opacity: 1→0, scale: 1→0.95` | Same as enter | Wrap in `AnimatePresence mode="popLayout"` |
| GroupCard (hover) | `scale: 1→1.02` | `stiffness: 400, damping: 17` | Desktop only via `whileHover` |
| GroupCard (press) | `scale: 1→0.98` | `stiffness: 400, damping: 17` | Mobile via `whileTap` |
| Streaming dots | `opacity: [0.3, 1, 0.3]` loop | `duration: 1.2, repeat: Infinity` | Stagger: `delay: i * 0.2` |
| Message list (layout) | `layout` prop on message items | Automatic | Only on items that change position |

### `prefers-reduced-motion` Policy

- Disable all spring/transform animations
- Keep opacity fades (accessibility-safe)
- Streaming dots: static (all visible at full opacity)

## Mobile-First Layout Contract

### Viewport Strategy

| Property | Value |
|----------|-------|
| Minimum width | 320px (iPhone SE 1st gen) |
| Full height | `100dvh` (dynamic viewport height — accounts for mobile browser chrome) |
| Safe areas | `env(safe-area-inset-bottom)` on chat input container |
| Orientation | Portrait primary, landscape functional but not optimized |

### Breakpoint Behavior

| Breakpoint | Chat Layout Changes |
|------------|-------------------|
| 320px (base) | Fullscreen chat, no padding. Cards full-width. Header 48px. Input fixed bottom |
| 640px (sm) | 8px side padding on message area |
| 768px (md) | 16px side padding. Max-width 640px on message area, centered |
| 1024px (lg) | Max-width 768px on message area. Cards may show 2-column layout if space allows |

### Virtual Keyboard Handling

- Listen to `window.visualViewport.resize` event
- On keyboard open: scroll message list to bottom smoothly
- Chat input stays fixed at bottom — keyboard pushes viewport, not the input

### Header (48px height on all viewports)

| Element | Specification |
|---------|---------------|
| Left | App name "Aja Agora" in heading weight (600), 16px size |
| Right | Minimal — "Nova conversa" icon button (RefreshCw icon from Lucide) |
| Background | `--card` with bottom border `--border` |
| Position | Sticky top |

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Color contrast | All text meets WCAG 2.1 AA (4.5:1 for body, 3:1 for large text). Teal primary on white must pass |
| Focus indicators | Visible focus ring (`--ring` token) on all interactive elements. 2px offset |
| Screen reader | Chat messages use `role="log"` with `aria-live="polite"`. New messages announced. Artifacts have descriptive `aria-label` |
| Keyboard nav | Tab through messages and artifact actions. Enter to activate GroupCard. Escape to dismiss tooltips |
| Reduced motion | Respect `prefers-reduced-motion` — opacity-only transitions, no springs |
| Touch targets | Minimum 44x44px for all interactive elements |
| Labels | Send button has `aria-label="Enviar mensagem"`. Input has `aria-label="Digite sua mensagem"` |

## State Contract (Zustand Store)

This section defines the state shape that the chat UI depends on, ensuring the executor implements the correct contract.

```typescript
interface ChatMessage {
  id: string;                    // crypto.randomUUID()
  role: 'user' | 'assistant';
  content: string;               // Accumulated text (streaming) or full text (user)
  artifacts: Artifact[];         // Inline artifacts attached to this message
  createdAt: Date;
  status: 'pending' | 'streaming' | 'complete' | 'error';
}

interface Artifact {
  id: string;
  type: 'group_card' | 'comparison_table' | 'simulation_result';
  payload: GroupCardPayload | ComparisonTablePayload | SimulationResultPayload;
}

interface ChatState {
  conversationId: string | null;
  messages: ChatMessage[];
  isStreaming: boolean;
  error: string | null;
  // Actions
  sendMessage: (content: string) => Promise<void>;
  retry: () => void;
  reset: () => void;
}
```
