|----------|-------------|
| client -> /api/leads | Untrusted PII input crosses from browser to server |
| client -> /api/chat | Chat messages cross from browser to agent |
| agent -> leads DB | capture_lead tool writes PII to database |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | /api/leads | mitigate | Validate all input with Zod leadSchema on server side; verify conversationId exists in DB |
| T-05-02 | Information Disclosure | artifact payload | mitigate | LeadFormPayload contains NO PII fields; PII flows only to /api/leads and leads table |
| T-05-03 | Denial of Service | /api/leads | mitigate | Rate limit on /api/leads endpoint using existing checkRateLimit |
| T-05-04 | Spoofing | /api/leads | accept | No auth required (anonymous users); conversationId validation is sufficient for MVP |
| T-05-05 | Information Disclosure | chat messages | mitigate | sendMessage("Dados enviados com sucesso") contains no PII; form data never enters message content |
</threat_model>
