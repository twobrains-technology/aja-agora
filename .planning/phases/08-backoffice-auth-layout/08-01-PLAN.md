**Step 1: Create route protection proxy** at `src/proxy.ts` per D-06, BSEC-01.

Next.js 16 uses `proxy.ts` in `src/` (since this project uses `src/` directory). Create:

```typescript
// src/proxy.ts
import { NextRequest, NextResponse } from "next/server";
import { auth } from "@/lib/auth";
import { headers } from "next/headers";

export async function proxy(request: NextRequest) {
  const session = await auth.api.getSession({
    headers: await headers(),
  });

  if (!session) {
    return NextResponse.redirect(new URL("/admin/login", request.url));
  }

  return NextResponse.next();
}

export const config = {
  matcher: ["/admin/((?!login).*)"],
};
```

**NOTE:** If `proxy.ts` in `src/` doesn't work (Next.js 16 may expect it at project root), move to project root `/proxy.ts`. Check Next.js 16 docs on proxy file location. The research says project root. Try `src/proxy.ts` first since the project uses `src/` app directory.

**Step 2: Create admin seed script** at `src/scripts/seed-admin.ts` per D-18:

```typescript
// src/scripts/seed-admin.ts
// Run with: npx tsx src/scripts/seed-admin.ts
import { auth } from "../lib/auth";
import { db } from "../db";
import { user } from "../db/schema";
import { eq } from "drizzle-orm";

async function seedAdmin() {
  const email = process.env.ADMIN_EMAIL;
  const password = process.env.ADMIN_PASSWORD;

  if (!email || !password) {
    console.error("ADMIN_EMAIL and ADMIN_PASSWORD environment variables are required");
    process.exit(1);
  }

  try {
    // Create user via Better Auth API
    const result = await auth.api.signUpEmail({
      body: {
        email,
        password,
        name: "Admin",
      },
    });

    // Update role to admin (input: false prevents setting during signup)
    await db
      .update(user)
      .set({ role: "admin" })
      .where(eq(user.id, result.user.id));

    console.log(`Admin user created successfully: ${email} (role: admin)`);
  } catch (error) {
    if (error instanceof Error && error.message.includes("already exists")) {
      console.log(`Admin user already exists: ${email}`);
    } else {
      console.error("Failed to create admin user:", error);
      process.exit(1);
    }
  }
}

seedAdmin();
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> /api/auth/* | Login credentials cross from untrusted client to auth handler |
| Browser -> /admin/* | Session cookie validates admin access |
| proxy.ts -> auth.api | Session validation before route access |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Spoofing | /api/auth/sign-in | mitigate | Better Auth handles password hashing with timing-safe comparison; rate limiting via existing rate-limit middleware on /api/auth/* |
| T-08-02 | Tampering | user.role field | mitigate | `input: false` on role additionalField prevents self-assignment during signup/update |
| T-08-03 | Spoofing | proxy.ts session check | mitigate | Use `auth.api.getSession()` (DB-validated), NOT `getSessionCookie()` (cookie-only) |
| T-08-04 | Elevation of Privilege | seed-admin.ts | accept | Script requires env vars and direct server access; not exposed via API |
| T-08-05 | Spoofing | CSRF on auth endpoints | mitigate | Better Auth includes CSRF protection by default |
| T-08-06 | Information Disclosure | session.user data | mitigate | Only role field added to session; no sensitive data exposed |
</threat_model>
