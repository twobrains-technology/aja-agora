### Task 4: Create the MessageList with auto-scroll

<action>
Create `src/components/chat/message-list.tsx`:

Scrollable container for chat messages with auto-scroll behavior:

Component must:
- Accept `{ messages: ChatMessage[], isStreaming: boolean }` props
- Render each message using `ChatMessage` component
- Use a `ref` on the scroll container
- Implement auto-scroll logic:
  - Use an `IntersectionObserver` on a sentinel `<div>` at the bottom
  - Track `isAtBottom` state
  - When `isAtBottom === true` and new messages arrive, scroll to bottom
  - When `isAtBottom === false` (user scrolled up), do NOT auto-scroll
  - During streaming (`isStreaming === true`), always auto-scroll to show latest content
- Show a "scroll to bottom" button when user has scrolled up and new messages are below
- Empty state: when no messages, show a welcome message like "Ola! Diga-me o que voce deseja — um carro, uma casa, um servico — e eu encontro o melhor consorcio para voce."
- Add bottom padding sufficient to clear the fixed input area
- Mark as `"use client"`

Auto-scroll implementation uses `useEffect` watching `messages` and `isStreaming`:
```typescript
useEffect(() => {
  if (isAtBottom || isStreaming) {
    sentinelRef.current?.scrollIntoView({ behavior: isStreaming ? "auto" : "smooth" });
  }
}, [messages, isStreaming, isAtBottom]);
```

Use `behavior: "auto"` during streaming (instant scroll to keep up with tokens) and `behavior: "smooth"` for new messages (user-initiated).
</action>

<threat_model>
## Threat Model (ASVS L1)
| Threat | Severity | Mitigation |
|--------|----------|------------|
| XSS via user message reflection | High | React auto-escapes text content. Never render user messages as HTML. `whitespace-pre-wrap` preserves formatting without HTML interpretation. |
| Input flood (rapid message sending) | Medium | `isStreaming` guard prevents concurrent sends. Backend rate limiting (from Phase 2) adds server-side protection. |
| Virtual keyboard layout breakage on mobile | Medium | `dvh` units and `visualViewport` API handle keyboard appearance. Fixed bottom input uses `env(safe-area-inset-bottom)`. Test on iOS Safari and Chrome Android. |
| Large message history degrading scroll performance | Low | For MVP, 50-100 messages are fine without virtualization. If needed later, add windowing library. Auto-scroll uses efficient IntersectionObserver. |
| Missing chat-input focus on mobile return from background | Low | `autoFocus` on textarea. `useEffect` with `focus()` on mount. Mobile browsers may override — acceptable for MVP. |
</threat_model>
