**Step 2: Create `src/lib/admin/dashboard-queries.ts`.**

Import `db` from `@/db`, `leads`, `conversations`, `leadEvents` from `@/db/schema`, Drizzle operators (`sql`, `count`, `eq`, `gte`, `lte`, `and`), `FUNNEL_STAGES` from `./dashboard-types`, and date-fns functions (`subDays`, `differenceInDays`, `startOfDay`, `format`).

Implement 4 exported async functions, all accepting `fromDate: Date` and `toDate: Date`:

**`computeKpis(fromDate, toDate)`** returns `KpiData` (per D-04, D-05, D-06):
1. Calculate period length via `differenceInDays(toDate, fromDate)`.
2. Compute previous period dates: `prevFrom = subDays(fromDate, periodLength)`, `prevTo = subDays(toDate, periodLength)`.
3. Run 4 queries in `Promise.all` for current period:
   - `totalLeads`: `SELECT COUNT(*) FROM leads WHERE created_at BETWEEN fromDate AND toDate`
   - `leadsToday`: `SELECT COUNT(*) FROM leads WHERE created_at >= (NOW() AT TIME ZONE 'America/Sao_Paulo')::date` — hardcode timezone per research finding #4
   - `avgFunnelDays`: Use raw SQL — `SELECT ROUND(AVG(EXTRACT(EPOCH FROM (le.created_at - l.created_at)) / 86400), 1) FROM leads l JOIN lead_events le ON le.lead_id = l.id WHERE l.created_at BETWEEN fromDate AND toDate AND le.to_stage IN ('fechado_ganho', 'perdido')`. If null (no completed leads), return 0.
   - `conversionRate`: `SELECT COUNT(*) FILTER (WHERE stage = 'fechado_ganho') * 100.0 / NULLIF(COUNT(*), 0) FROM leads WHERE created_at BETWEEN fromDate AND toDate`. If null, return 0.
4. Run same totalLeads + leadsToday + avgFunnelDays + conversionRate for previous period.
5. Compute trend percentages: `prevValue === 0 ? (currentValue > 0 ? 100 : 0) : Math.round(((currentValue - prevValue) / prevValue) * 100)`.
6. For "leadsToday" trend, compare vs yesterday's count for the same timezone.

**`computeFunnelStages(fromDate, toDate)`** returns `FunnelStage[]` (per D-07, D-08):
1. Query `SELECT stage, COUNT(*) FROM leads WHERE created_at BETWEEN fromDate AND toDate GROUP BY stage`.
2. Map results into FUNNEL_STAGES order (excluding "perdido" per research finding #2).
3. For each stage: compute `percentOfTotal = (count / firstStageCount) * 100` (first stage is always 100%).
4. For each stage after first: compute `dropOffRate = ((prevCount - count) / prevCount) * 100`. First stage dropOffRate = 0.
5. If a stage has 0 leads, still include it with count=0.

**`computeDailyVolume(fromDate, toDate)`** returns `DailyVolume[]` (per D-11):
1. Query: `SELECT DATE(created_at AT TIME ZONE 'America/Sao_Paulo') as date, COUNT(*) FROM leads WHERE created_at BETWEEN fromDate AND toDate GROUP BY date ORDER BY date`.
2. Fill gaps: iterate from fromDate to toDate day-by-day, inserting `{ date: "YYYY-MM-DD", count: 0 }` for days with no leads. This prevents chart gaps.

**`computeChannelBreakdown(fromDate, toDate)`** returns `ChannelBreakdown[]` (per D-14, D-16):
1. Query: `SELECT c.channel, COUNT(*) FROM leads l JOIN conversations c ON l.conversation_id = c.id WHERE l.created_at BETWEEN fromDate AND toDate GROUP BY c.channel`.
2. Compute `percent = (count / totalCount) * 100` for each channel.
3. If a channel has 0, still include it with count=0, percent=0.
